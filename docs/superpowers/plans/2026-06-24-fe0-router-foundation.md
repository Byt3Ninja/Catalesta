# FE-0: Router + Console Shell Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hand-rolled regex route resolver in `App.tsx` with `react-router-dom`, preserving every current routing behavior exactly.

**Architecture:** Wrap the app in `<BrowserRouter>` and express the existing routes as a `<Routes>` tree. Public routes render their page directly; console routes keep the existing render-prop `ConsoleGate` unchanged. Route params come from `useParams()` via thin wrapper components that pass the existing props into the existing page components — page internals and their tests are untouched.

**Tech Stack:** React 19, react-router-dom v7, @tanstack/react-query, Vitest + Testing Library, Vite.

## Global Constraints

- Behavior-preserving migration only — no redesign, no new screens, no new flows.
- `react-router-dom` pinned to v7 (current stable).
- `DirectionProvider` stays at the root in `main.tsx`, outside the router — do not move it.
- `/apply/:cohortId` and `/health` stay **public** (no gate), exactly as today.
- Do NOT double-decode params: react-router already URL-decodes `useParams()` values, so the old `decodeURIComponent(...)` calls must be dropped, not carried over.
- Pages keep their own `AppShell` — do NOT add a second global shell/nav (see Contradiction note).
- Frontend routing is never authorization: the server-side gate (`ConsoleGate` → session/org queries) remains the only thing that admits a console surface.

## Contradiction Resolved (2026-06-24)

The approved spec (`docs/superpowers/specs/2026-06-24-phase1a-frontend-router-foundation-design.md`) called for a `ConsoleLayout` nav shell around an `<Outlet/>` plus org-via-context (`useCurrentOrganization()`). On reading the code, every console page already renders its **own** `AppShell` (e.g. `HomePage` does `<AppShell rail={<nav aria-label="Sections">Home</nav>}>`), so a second global shell would duplicate the rail — contradicting the spec's own "behavior-preserving — no redesign" mandate and rule 08 ("don't duplicate / no second design system"). **Decision (user-approved):** FE-0 ships the pure router migration only; the render-prop `ConsoleGate` is kept; no `ConsoleLayout`, no org context, no nav shell. The unified shell is deferred to its own slice (FE-0.5) where the four per-page `AppShell` rails are consolidated deliberately, with its own tests.

Replace the contents of `frontend/src/app/App.tsx` from the `/** No router is installed ... */` block (line 26) through the end of the file with the following. **Keep lines 1–24 (imports) and the `ConsoleGate` function (lines 40–123) exactly as they are**, then add `import { BrowserRouter, Routes, Route, useParams } from 'react-router-dom'` to the import block at the top.

```tsx
// --- Route elements -------------------------------------------------------
// Thin wrappers read params from react-router (already URL-decoded — no manual
// decodeURIComponent) and pass the existing props into the existing pages.
// Behavior matches the previous regex resolver one-for-one.

function ApplyRoute() {
  const { cohortId } = useParams()
  return <ApplyPage cohortId={cohortId!} />
}

function ProgramsRoute() {
  return <ConsoleGate>{(org) => <ProgramsPage organization={org} />}</ConsoleGate>
}

function SubmissionsRoute() {
  const { cohortId } = useParams()
  return (
    <ConsoleGate>{(org) => <SubmissionsPage cohortId={cohortId!} organization={org} />}</ConsoleGate>
  )
}

function SubmissionDetailRoute() {
  const { cohortId, submissionId } = useParams()
  return (
    <ConsoleGate>
      {(org) => (
        <SubmissionDetailPage cohortId={cohortId!} submissionId={submissionId!} organization={org} />
      )}
    </ConsoleGate>
  )
}

// Root and any unknown console/onboarding path → the gate decides. Home is the
// consent-aware surface, so it renders inside the ConsentProvider seam (FR-006).
function HomeRoute() {
  return (
    <ConsoleGate>
      {(org) => (
        <ConsentProvider>
          <HomePage organization={org} />
        </ConsentProvider>
      )}
    </ConsoleGate>
  )
}

export function AppRoutes() {
  return (
    <Routes>
      {/* Public — render directly, no gate. */}
      <Route path="/health" element={<HealthPage />} />
      <Route path="/login" element={<LoginPage />} />
      <Route path="/register" element={<RegisterPage />} />
      <Route path="/forgot-password" element={<ForgotPasswordPage />} />
      <Route path="/auth/callback" element={<AuthCallbackPage />} />
      <Route path="/auth/reset-password" element={<ResetPasswordPage />} />
      <Route path="/auth/email-verified" element={<EmailVerifiedPage />} />
      <Route path="/apply/:cohortId" element={<ApplyRoute />} />

      {/* Console — gated by ConsoleGate (server-side session/org). */}
      <Route path="/programs" element={<ProgramsRoute />} />
      <Route
        path="/cohorts/:cohortId/submissions/:submissionId"
        element={<SubmissionDetailRoute />}
      />
      <Route path="/cohorts/:cohortId/submissions" element={<SubmissionsRoute />} />

      {/* Root and any other route → gate decides (today's fallthrough). */}
      <Route path="/" element={<HomeRoute />} />
      <Route path="*" element={<HomeRoute />} />
    </Routes>
  )
}

export function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <BrowserRouter>
        <AppRoutes />
      </BrowserRouter>
    </QueryClientProvider>
  )
}
```
