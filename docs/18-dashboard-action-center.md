# Dashboard and Action Center

## Default Dashboard Priority

1. Required actions
2. Deadlines
3. Current stage
4. Upcoming sessions
5. Blocked items
6. Recent decisions
7. Progress
8. Relevant opportunities

## Action Card Structure

Every action card must show:

- what is required
- why it matters
- deadline
- responsible person
- direct action link
- blocking dependency

## Role Examples

### Founder

- complete missing application section
- upload rejected document
- confirm mentor session
- complete training assignment

### Mentor

- accept assignment
- prepare for session
- submit session notes
- review startup task

### Evaluator

- complete assigned evaluation
- resolve conflict declaration
- review updated application

### Program Manager

- review delayed applications
- assign evaluators
- approve stage transition
- resolve missing role coverage
