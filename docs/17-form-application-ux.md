# Form and Application UX

## Requirements

- autosave
- explicit save status
- resume from last section
- section progress
- readiness summary
- inline validation
- conditional questions
- repeatable sections
- file requirements before upload
- no data loss after errors
- Arabic and English parity
- keyboard-only completion
- screen-reader support

## Profile Reuse

Display:

- imported Startup Gate data
- missing required data
- editable shared data
- program-only data
- fields that will create update proposals
- immutable submission snapshot behavior

## Submission Readiness

Before submission show:

- incomplete required fields
- missing documents
- unresolved validation errors
- required declarations
- consent status

## Long Forms

Split by logical task, not database structure.

Each section should have:

- title
- purpose
- completion state
- save state
- clear next step
