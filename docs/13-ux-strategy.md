# UX Strategy

## Objective

Design the portal so users complete tasks correctly, quickly, and with minimal cognitive load.

Do not optimize for raw session duration. Optimize for:

- task success
- low error rates
- low abandonment
- low time-to-complete
- strong return behavior for recurring work
- low support dependency
- clear next actions

## UX Principles

1. Role-based experiences
2. Task-oriented navigation
3. One primary action per screen
4. Progressive disclosure
5. Save-and-resume for long flows
6. Reuse Startup Gate profile data
7. Show clear status and next action
8. Avoid duplicate data entry
9. Support Arabic and English equally
10. Meet WCAG 2.2 AA
11. Design mobile around essential actions
12. Validate usability with real tasks

## Primary Role Workspaces

- Founder
- Co-Founder
- Mentor
- Trainer
- Evaluator
- Judge
- Service Provider
- Program Manager
- Program Coordinator
- Organization Administrator

Each workspace should expose only relevant navigation, actions, and data.

- Reports

## Default Home Experience

Every role lands on an Action Center showing:

1. urgent required actions
2. approaching deadlines
3. current stage or status
4. upcoming sessions
5. recent decisions
6. blocked items
7. recommended opportunities

## Critical UX Metrics

- task-success rate
- application completion rate
- abandonment by section
- validation-error rate
- time to complete application
- time to configure a program
- evaluation completion time
- session confirmation rate
- training completion rate
- search success rate
- support requests per workflow
- returning active users
