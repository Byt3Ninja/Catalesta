# Design System

> Owner: UX · Last-updated: 2026-06-19 · Source-of-truth: docs/product/scope-register.md (scope), docs/plan/roadmap.md (sequence)

## Scope

The design system must define:

- typography
- spacing
- grid
- breakpoints
- color tokens
- semantic status colors
- elevation
- borders
- iconography
- motion
- focus states
- RTL behavior

## Required Components

- buttons
- inputs
- selects
- multi-selects
- date pickers
- file uploads
- tables
- cards
- tabs
- side navigation
- top navigation
- stepper
- timeline
- badges
- alerts
- dialogs
- drawers
- empty states
- loading states
- error states
- permission-denied states
- pagination
- filters
- search
- charts
- action center
- activity feed
- profile completeness indicator

## Storybook Requirements

Each component must include:

- all states
- all variants
- accessibility notes
- RTL example
- mobile example
- usage guidance
- misuse examples

## Interaction Rules

- One dominant primary action
- Destructive actions require confirmation
- Disabled actions explain why
- Validation appears near the field
- Loading states preserve layout
- Status never depends on color alone
